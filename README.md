# ETER — статичен IPTV плейър (M3U)

Лек, изцяло клиентски (HTML/CSS/JS) плейър за M3U/M3U8 плейлисти. Без бекенд, без база данни — качваш на GitHub Pages и работи.

⚠️ **Важно:** тази технология е неутрална — играе всякакви HLS/MP4 линкове, които сложиш в плейлиста. Слагай само съдържание, което имаш право да препредаваш (свои стриймове, лицензирани/публични източници). Примерният `playlist.m3u` съдържа само публични demo/test стриймове (Apple, Mux, Blender Foundation), за да работи сайтът "из кутията".

## Структура на проекта

```
index.html        — страницата (плейър + списък с канали)
style.css         — визия
app.js            — парсер на M3U + логика на плейъра (hls.js)
playlist.m3u      — твоят основен списък с канали (РЕДАКТИРАШ ТОЗИ ФАЙЛ)
playlists.json    — регистър на всички плейлисти (за бутона "Плейлист" горе)
playlists/        — папка за допълнителни .m3u файлове (по категория, доставчик и т.н.)
```

## Как да обновяваш списъка с канали (най-лесният начин)

Отвори `playlist.m3u` с обикновен текстов редактор и добави канал по този образец:

```
#EXTINF:-1 tvg-logo="https://example.com/logo.png" group-title="Национални",Име на канала
https://example.com/stream.m3u8
```

- `group-title` — определя в коя секция/група ще се появи каналът в страничния списък
- `tvg-logo` — линк към лого (PNG/JPG); ако няма, ще се покажат първите 2 букви от името
- Реда **след** `#EXTINF` трябва да е директно линкът към стрийма (`.m3u8` за HLS, или `.mp4`/друг формат)

Запазваш файла → `git add . && git commit -m "update channels" && git push` → готово, GitHub Pages автоматично обновява сайта (обикновено за под минута).

Никакъв код не се пипа за да добавиш/махнеш/преименуваш канал.

## Множество плейлисти (по желание)

Ако искаш отделни списъци (например "Новини", "Спорт", "Детски"), които да се превключват с падащото меню "Плейлист" горе:

1. Сложи нов файл, напр. `playlists/sport.m3u`, в същия M3U формат.
2. Добави запис в `playlists.json`:

```json
[
  { "name": "Моят плейлист", "file": "playlist.m3u" },
  { "name": "Спорт", "file": "playlists/sport.m3u" }
]
```

Менюто ще се обнови автоматично при следващото зареждане на сайта.

## Други начини за зареждане на списък (без да пипаш репото)

- **"+ URL"** бутон горе — въвеждаш директен линк към чужд/твой M3U файл, хостван някъде онлайн (трябва източникът да позволява CORS, иначе браузърът ще го блокира).
- **"+ Файл"** бутон — зарежда M3U файл директно от компютъра ти (само за тестово гледане в текущата сесия, не се записва в репото).

## Локално тестване

Понеже сайтът прави `fetch()` за `playlist.m3u` и `playlists.json`, трябва да го пуснеш през локален сървър (не просто като `file://`), например:

```bash
cd eter-iptv
python3 -m http.server 8000
# после отвори http://localhost:8000
```

## Качване в GitHub Pages

1. Създай ново repo в GitHub и качи всички файлове от тази папка в основната директория (или в `/docs`, ако предпочиташ).
2. В repo → **Settings → Pages** → избери branch (`main`) и папка (`/` или `/docs`).
3. Изчакай 1-2 минути — сайтът ще е достъпен на `https://<потребител>.github.io/<repo>/`.

## Поддържани типове стрийм

- **HLS (`.m3u8`)** — през [hls.js](https://github.com/video-dev/hls.js)
- **Суров MPEG-TS (live, без `.m3u8`)** — типично за `udpxy`-стил линкове (`http://ip:port/udp/239.x.x.x:port`) — през [mpegts.js](https://github.com/xqq/mpegts.js)
- **Директни файлове** (`.mp4`, `.webm` и др.) — нативно през `<video>`

## ⚠️ HTTPS сайт + HTTP стриймове = Mixed Content блокиране

Ако хостваш сайта на GitHub Pages (винаги HTTPS), а някой от твоите стриймове е чист `http://` (типично за домашни/ISP IPTV gateway-та), браузърът **ще блокира** зареждането му — нарича се **Mixed Content** и е защита на самия браузър, не бъг в кода, и **не може да се заобиколи с JavaScript**.

Решение: проксирай тези линкове през безплатен **Cloudflare Worker**, който ги "облича" в HTTPS.

### 1. Деплой на прокси-то (веднъж)

1. [dash.cloudflare.com](https://dash.cloudflare.com) → регистрация (безплатно)
2. **Workers & Pages** → **Create** → **Create Worker**
3. Дай му име (напр. `iptv-proxy`) → **Deploy**
4. **Edit code** → изтрий примерния код → постави съдържанието на `proxy/worker.js` от този проект
5. В горната част на файла, в `ALLOWED_HOSTS`, провери че е сложен `ip:port`-а на твоя IPTV източник (по подразбиране е сложен `45.84.187.172:3001` — само хостове в този списък могат да се проксират, заради сигурност)
6. **Save and Deploy**
7. Копирай адреса на Worker-а, нещо като `https://iptv-proxy.твое-потребителско-име.workers.dev`

### 2. Преобразуване на плейлиста (всеки път, когато добавиш нови HTTP линкове)

```bash
node tools/proxify.js playlist.m3u playlist.m3u https://iptv-proxy.твое-потребителско-име.workers.dev/
```

Това минава през целия `playlist.m3u`, заменя само `http://` линковете (HTTPS остават недокоснати) с проксирана версия, и пише резултата обратно. Виждаш и обобщение колко линка са преобразувани.

Push-ваш промените → готово.

### Важно за надеждност

- Ако IPTV gateway-я филтрира по IP (приема връзки само от мрежата на конкретен доставчик), Cloudflare Worker-ите се изпълняват от сървъри с различни IP-та — прокси-то няма да помогне в този случай. Тествай директно линка във VLC първо, за да изключиш този проблем.
- Безплатният план на Cloudflare Workers позволява 100 000 заявки/ден — напълно достатъчно за лична употреба.



- HLS (`.m3u8`) се възпроизвежда през [hls.js](https://github.com/video-dev/hls.js) (зареден от CDN); Safari ползва вградената си HLS поддръжка.
- Други формати (`.mp4` и т.н.) се пускат директно през нативния `<video>` таг.
- Последно гледаният плейлист се помни в `localStorage`.
- Ако лого-линк не зареди (счупен URL), автоматично пада на текстов placeholder с инициалите на канала.
